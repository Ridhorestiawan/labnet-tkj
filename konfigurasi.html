Konfigurasi Jaringan

Apa itu Konfigurasi Jaringan?

Konfigurasi jaringan adalah proses pengaturan parameter jaringan pada perangkat agar dapat terhubung dan berkomunikasi dalam suatu jaringan. Konfigurasi meliputi pengaturan alamat IP, subnet mask, gateway default, dan DNS server.

1. IP Address

IP Address (Internet Protocol Address) adalah alamat unik yang diberikan pada setiap perangkat dalam jaringan. IP address terdiri dari 4 blok angka (0-255) yang dipisahkan oleh titik, misalnya 192.168.1.11.

Dalam jaringan LAN laboratorium, IP address dibagi menjadi dua kategori:

  • IP Static — Alamat tetap yang dikonfigurasi manual pada setiap perangkat. Biasanya digunakan untuk server, printer jaringan, dan perangkat yang membutuhkan alamat tetap.
  • IP DHCP — Alamat diberikan secara otomatis oleh router/switch DHCP server. Cocok untuk perangkat klien yang tidak memerlukan alamat tetap.

2. Subnet Mask

Subnet Mask berfungsi untuk membedakan bagian network ID dan host ID dalam sebuah IP address. Dalam jaringan lab, subnet mask yang digunakan adalah 255.255.255.0 (prefix /24).

Contoh: IP 192.168.1.11 dengan subnet 255.255.255.0 berarti:

  • Network ID: 192.168.1.0
  • Host ID range: 192.168.1.1 hingga 192.168.1.254
  • Broadcast: 192.168.1.255

3. Gateway

Gateway adalah perangkat yang menghubungkan satu jaringan dengan jaringan lain. Dalam laboratorium, gateway biasanya adalah 192.168.1.1 (alamat router). Semua perangkat yang ingin mengakses internet harus melalui gateway ini.

5. Cara Setting IP Static (Windows)

Berikut langkah-langkah konfigurasi IP address static pada sistem operasi Windows:

  1. Buka Control Panel → Network and Sharing Center.
  2. Klik Change adapter settings di menu sebelah kiri.
  3. Klik kanan pada jaringan yang digunakan (Ethernet / Local Area Connection), pilih Properties.
  4. Pilih Internet Protocol Version 4 (TCP/IPv4), lalu klik Properties.
  5. Pilih Use the following IP address.
  6. Isi IP address, subnet mask, dan default gateway sesuai tabel di atas.
  7. Isi DNS server (misal: 8.8.8.8 untuk Google DNS).
  8. Klik OK untuk menyimpan konfigurasi.

6. Perintah Dasar Troubleshooting Jaringan

Berikut perintah dasar yang dapat digunakan untuk mengecek koneksi jaringan:

ipconfig — Menampilkan konfigurasi IP komputer saat ini.

C:\Users\Lab> ipconfig

Ethernet adapter Local Area Connection:
   IP Address. . . . . . . . . : 192.168.1.11
   Subnet Mask . . . . . . . . : 255.255.255.0
   Default Gateway . . . . . . : 192.168.1.1

ping — Mengecek koneksi ke perangkat lain.

C:\Users\Lab> ping 192.168.1.1

Reply from 192.168.1.1: bytes=32 time<1ms TTL=128
Reply from 192.168.1.1: bytes=32 time<1ms TTL=128
Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)

tracert — Melacak rute paket data ke tujuan.

C:\Users\Lab> tracert google.com

1 <1ms <1ms <1ms 192.168.1.1
2 10ms 8ms 9ms 10.0.0.1
3 25ms 22ms 24ms 203.xxx.xxx.xxx

nslookup — Mengecek resolusi DNS.

C:\Users\Lab> nslookup google.com

Server: dns.google
Address: 8.8.8.8

Non-authoritative answer:
Name: google.com
Addresses: 142.250.xxx.xxx
